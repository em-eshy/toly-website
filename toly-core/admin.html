How real publishing works

Static sites need data updates.

A browser on GitHub Pages cannot safely write to the repository by itself. The private CLI console can talk to a local backend now; a production version would still need authentication before it touched public data.

CLI option

python scripts/publish_log.py --title "Weekly note" --summary "Short public summary" --category company --body "Longer update"

The CLI writes to data/news.json. Commit that file when the update is approved for the public site.

The editor is private. Public visitors should only see approved updates on the News page.
